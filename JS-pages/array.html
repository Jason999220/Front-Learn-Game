新增資料

JS
content
run

為什麼要新增資料呢？因為我們不可能一開始就把資料寫死，所以才會有新增資料的方法。
網路上有許多方法，這邊就提幾個比較常用的。
1. push => 從後面新增，會回傳更新後長度。
2. unshift => 從前面新增，會回傳更新後長度。
3. concat => 從後面新增，會回傳更新後陣列，保留原始陣列。
4. Spread Operator => 會回傳更新後陣列，這個比較特殊下一篇會好好講解。

可以嘗試在左側練習每個不同的方法，了解其中的差異，一定有特殊用途才會衍伸出這麼多種方法。

Spread Operator & Rest Operator

JS
content
run

【...】 是一個極其強大的功能，展開陣列取得陣列內的資料，或合併陣列等功能。
這裡會統一講述 Spread Operator & Rest Operator 其實兩者都是以【...】當語法。

Spread Operator => 展開運算子 ，簡而言之就像左側的字串分離、合併陣列，將其內容物展開。
Rest Operator => 其餘運算子 ，簡而言之就像左側的結構賦值、函式參數，取得其餘內容物

淺拷貝是拷貝的某一種方式，這個比較複雜會在稍後說明。
